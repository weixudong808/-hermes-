## 二、问卷→服务配置映射

```
会员填完问卷（9项）
    │
    ├─ 所有群通用（不受问卷控制）：
    │   S1  训练记录 / S2  数据查询 / S8  日常互动
    │   S9  知识问答 / S10 隐私保护 / S12 教练权限
    │   S13 跨群隔离 / S14 会员不记录
    │
    ├─ 第1项（称呼）→ 写入 profile.json + group_map.json（本地存储）
    │
    ├─ 第2项（目标：增肌/减脂/塑形）→ S3/S4 报告的情绪价值方向
    │
    ├─ 第4-6项（吃饭时间）→ S5 饮食提醒 cron job
    │
    ├─ 第7项（提醒频率）→ S5 细节 + S6
    │
    ├─ 第8项（报告开关）→ S3 + S4
    │
    └─ 第9项（风格）→ S7
```

## 三、报告情绪价值规则（S3/S4）

详见 SKILL.md 第四节 4.0。

核心：生成报告前检查训练记录数量 → 缺席发关怀（不发数据）→ 活跃发亮点+专业鼓励 → 偶尔发肯定+温和鼓励。
缺席关怀参考五月天阿信风格：温柔、不施压、共情、"我一直在这里等你"。
绝对禁止指责、施压、制造焦虑。

## 四、Cron Job 触发机制

详见 SKILL.md 第十节 10.0。

核心：cron job 不依赖群消息触发。chat_id 在建群时写入 group_map.json，cron job 读 group_map 就知道往哪个群发。
每个"会员+提醒类型"是独立 cron job，deliver 指定到具体群。cron session 没有群聊上下文，prompt 必须自包含。
